About dcmdb Last updated 10/26/2014

This website was developed by Eric Bower. The mission of this service is to create a learning resource and development toolkit for developers and the curious.

Toolkit

As a student studying neuroscience, I found it rather difficult to find a common platform for Digital Imaging and Communications in Medicine (DICOM) tools to facilitate an understanding of how the most common, prominent diagnostic file operates. One of my goals with this project is to create a platform that at its core is a learning resource that provides tools and utilities to anyone interested in this technology.

Collaboration

If you are interested in helping, the source code is hosted on Github, feel free to have a look. Any suggestions, comments, or critcisms are greatly appreciated.

Open-Source all the way down

This project was developed in Python, the Flask framework, and Postgresql. The production server is hosted by Amazon on an Ubuntu EC2 virtualized server. It uses Gunicorn as the WSGI HTTP Server and NGINX as the Proxy HTTP Server.

Mudicom
I created mudicom to help facilitate the needs of this web service. I hope, in time, it will be good enough to replace all of the current python packages for DICOM files. It depends on the open-source C++ application for DICOM files: Grassroots DICOM
